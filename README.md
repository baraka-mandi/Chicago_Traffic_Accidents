
# Traffic Crash Severity Prediction

This project analyzes traffic crash data and builds machine learning models to predict crash severity. The project is developed using Jupyter Notebooks and includes a full workflow for data cleaning, visualization preparation, preprocessing, model training, and model evaluation.

The main prediction target is crash **SEVERITY**, grouped into three classes:

- **LOW**
- **MEDIUM**
- **HIGH**

The cleaned dataset is also prepared for dashboard visualization in **Google Data Studio / Looker Studio**.

---

## Project Objective

The goal of this project is to predict the severity level of traffic crashes using crash-related features such as weather condition, lighting condition, road condition, crash type, number of units involved, crash time, and injury information.

This project compares several machine learning models to determine which model performs best overall and which model is better at identifying **HIGH severity** crashes.

## Data Cleaning and Preparation

### Main Steps

The notebook:

1. Loads the original dataset:


```python
chunk_size = 90000
```

3. Drops unnecessary columns:

```python
cols_to_drop = ["CRASH_DATE_EST_I"]
df = df.drop(columns=cols_to_drop)
```

4. Removes rows with missing latitude values:

```python
df = df.dropna(subset=["LATITUDE"])
```

5. Creates a combined location column:

```python
df["LOCATION"] = df["LATITUDE"].astype(str) + "," + df["LONGITUDE"].astype(str)
```

6. Creates a simplified severity column:

```python
df["SEVERITY"] = df["MOST_SEVERE_INJURY"]
```

---

## Severity Mapping

The original `MOST_SEVERE_INJURY` values are converted into three severity classes.

| Original Injury Value    | New Severity Class |
| ------------------------ | ------------------ |
| NO INDICATION OF INJURY  | LOW                |
| REPORTED, NOT EVIDENT    | LOW                |
| NONINCAPACITATING INJURY | MEDIUM             |
| INCAPACITATING INJURY    | HIGH               |
| FATAL                    | HIGH               |

The mapping is performed using string replacement:

```python
df["SEVERITY"] = df["SEVERITY"].str.replace("NO INDICATION OF INJURY", "LOW", case=False)
df["SEVERITY"] = df["SEVERITY"].str.replace("REPORTED, NOT EVIDENT", "LOW", case=False)
df["SEVERITY"] = df["SEVERITY"].str.replace("NONINCAPACITATING INJURY", "MEDIUM", case=False)
df["SEVERITY"] = df["SEVERITY"].str.replace("INCAPACITATING INJURY", "HIGH", case=False)
df["SEVERITY"] = df["SEVERITY"].str.replace("FATAL", "HIGH", case=False)
```

## Missing Value Handling

The notebook handles missing values before training.

### Numerical Columns

Missing numerical values are filled with the column median:

```python
df[col] = df[col].fillna(df[col].median())
```

If an entire numerical column is missing, it is filled with `0`.

### Categorical Columns

Missing categorical values are filled with the mode:

```python
df[col] = df[col].fillna(df[col].mode()[0])
```

If an entire categorical column is missing, it is filled with:

```text
UNKNOWN
```

## Class Balancing

Traffic crash datasets are usually imbalanced, with many more LOW severity crashes than MEDIUM or HIGH severity crashes.

To reduce imbalance, the LOW severity class is undersampled:

```python
df_low_under = df_low.sample(
    n=min(200000, len(df_low)),
    random_state=42
)
```

The balanced dataset is created by combining:

```text
Undersampled LOW records
All MEDIUM records
All HIGH records
```

The final dataset is then shuffled:

```python
df_balanced = df_balanced.sample(
    frac=1,
    random_state=1
).reset_index(drop=True)
```

---

## Train, Validation, and Test Split

The balanced dataset is split into three parts:

| Dataset        | Percentage |
| -------------- | ---------: |
| Training Set   |        70% |
| Validation Set |        20% |
| Test Set       |        10% |

The split is stratified by `SEVERITY` so that the class distribution is preserved across training, validation, and testing datasets.

```python
train_df, temp_df = train_test_split(
    df_balanced,
    test_size=0.30,
    random_state=1,
    stratify=df_balanced[target]
)

val_df, test_df = train_test_split(
    temp_df,
    test_size=0.3333,
    random_state=1,
    stratify=temp_df[target]
)
```

## Models Trained

This project trains and compares five models.

---

### 1. Logistic Regression

Logistic Regression is used as a baseline model.

```python
LogisticRegression(
    class_weight="balanced",
    max_iter=1000,
    solver="lbfgs",
    random_state=42
)
```

This model uses balanced class weights to help handle class imbalance.

---

### 2. Random Forest Classifier

Random Forest is an ensemble tree-based model.

```python
RandomForestClassifier(
    n_estimators=100,
    class_weight="balanced",
    random_state=42,
    n_jobs=-1
)
```

It combines multiple decision trees and uses balanced class weights.

---

### 3. XGBoost Classifier

XGBoost is a gradient boosting model used for multiclass classification.

```python
XGBClassifier(
    n_estimators=300,
    learning_rate=0.05,
    max_depth=6,
    subsample=0.8,
    colsample_bytree=0.8,
    objective="multi:softmax",
    num_class=len(label_encoder.classes_),
    eval_metric="mlogloss",
    random_state=42,
    n_jobs=-1
)
```

The target labels are encoded using `LabelEncoder` before training XGBoost.

---

### 4. LightGBM Classifier

LightGBM is another gradient boosting model designed for efficiency and speed.

```python
LGBMClassifier(
```

## Results Interpretation

The **MLP Neural Network** achieved the highest test accuracy:

```text
0.9100
```

The **XGBoost model** also performed strongly in terms of test accuracy:

```text
0.9084
```

However, accuracy alone does not tell the full story. Since identifying HIGH severity crashes is important, the HIGH class recall and HIGH class F1 score should also be considered.

The **Logistic Regression** model had the best HIGH severity recall:

```text
0.53
```

The **LightGBM** model had the second-best HIGH severity recall:

```text
0.47
```

This means Logistic Regression and LightGBM were better at identifying HIGH severity crashes compared to Random Forest, XGBoost, and the MLP Neural Network.

Although the MLP Neural Network had the highest accuracy, it had low HIGH severity recall:

```text
0.06
```

This suggests that the MLP model performed well overall but struggled to correctly identify HIGH severity crashes.
